# F9: ConfigManager + 14 panes

## Files in scope (~5,884 LOC, 16 files)
ConfigManager.tsx (469), McpPane (535), PromptsTab (742), SettingsPane (497), HooksPane (371), SkillsEditor (356), PluginsPane (425), EnvVarsTab (302), RecordingPane (288), AgentEditor (230), MarkdownPane (179), SettingsTab (270), ThreePaneEditor (93), UnsavedTextEditors (84), EnvVarsReference (105), events.ts (1)

## Verified findings vs harness preamble

The preamble flagged four major issues. Code evidence both confirms and contradicts pieces:

1. **"PromptsTab regex helpers belong in lib/promptDiff.ts"** — already done. `applyRulesToText`, `generateRulesAndConflicts`, `classifyRule`, `diffLines`, `escapeRegex`, `unescapeRegex` are imported from `../../lib/promptDiff` at PromptsTab.tsx:6. Only 5-line `validatePattern` (L18-26) remains — UI-bound.

2. **"SettingsPane polls read_settings repeatedly"** — NOT borne out. `grep "setInterval" src/components/ConfigManager/SettingsPane.tsx` returns zero matches. SettingsPane reads file once via `useEffect` at L312. **The actual polling is at PromptsTab.tsx:282** — `setInterval(fetch, 2000)` calling `get_rule_match_counts` every 2 seconds.

3. **"15 panes each with 2-7 useState — state NOT lifted"** — confirmed. Total 92 `useState` calls across 12 files.

4. **"UnsavedTextEditors only registers panes that opt-in (silent data loss)"** — confirmed. HooksPane, PluginsPane, RecordingPane do NOT register. HooksPane's 7-field form is wiped on tab/CLI switch with no warning.

## Findings

### F9-1 — PromptsTab interval polling violates "no arbitrary polling intervals" rule
- **Category**: rule violation / event-driven design
- **Confidence**: high
- **Diff**: small (~30 LOC)
- **Location**: PromptsTab.tsx:273-284
- **Today**: `setInterval(fetch, 2000)` IPC every 2 seconds whether modal foreground or not. Match counts are emitted by Rust proxy when rule fires — they are event-driven, not state.
- **Proposed**: Drop interval. Subscribe via `listen<Record<string, number>>('rule_match_counts', ...)`. Backend: `app.emit_to(MAIN_WINDOW, "rule_match_counts", ...)` after `replace_all` runs.

### F9-2 — HooksPane, McpPane copy-mode, AgentEditor newName, SkillsEditor newName + name skip unsaved guard
- **Category**: silent data loss
- **Confidence**: high
- **Diff**: small (~40 LOC)
- **Location**: HooksPane.tsx (no `useUnsavedTextEditor` import); SkillsEditor `newName`; AgentEditor input
- **Today**: HooksPane keeps 7-field form wiped on tab switch / CLI switch / modal close with no warning. McpPane's `copyMode` and PluginsPane's `searchFilter` not registered.
- **Proposed**: Add `useUnsavedTextEditor` hook keyed off form-or-input identity, returning `{ title, before, after }` from snapshot of form fields. Each registration is 8-12 lines.

### F9-3 — Five panes duplicate `text/saved/loading/seedKey/textareaRef` quartet pattern
- **Category**: refactor / cohesion
- **Confidence**: high
- **Diff**: medium (-150/+120)
- **Today**: MarkdownPane, SettingsPane, EnvVarsTab inner pane, AgentEditor, SkillsEditor all repeat the same `text/saved/loading/seedKey/textareaRef` quartet, the same `load → setText → setSaved → setSeedKey` effect, the same `dirty = text !== saved` derivation, the same `Ctrl+S` handler, the same `useUnsavedTextEditor` registration. ~25-30 lines per pane × 5 panes is ~130 LOC duplication.
- **Proposed**: Extract `<TextFileEditor>` component:
  ```ts
  interface TextFileEditorProps {
    load: () => Promise<string>;
    save: (value: string) => Promise<void>;
    unsavedTitle: (scopeLabel: string) => string;
    scope: PaneComponentProps["scope"];
    cli: CliKind;
    projectDir: string;
    onStatus: (msg: StatusMessage | null) => void;
    highlight?: "json" | "toml" | "none";
    placeholder?: string;
    unsavedKeyPrefix: string;
    onTextChange?: (text: string) => void;
    preview?: boolean;
  }
  ```
  Owns the quartet, load effect, Ctrl+S, dirty, registration, syncScroll, highlight overlay, Save button. MarkdownPane, EnvVarsTab inner, SettingsPane core collapse to thin shells. AgentEditor/SkillsEditor have sidebar + different `selected` lifecycle — out of scope.
- **Risk**: NU-01 native-undo contract: must preserve `defaultValue + onInput + key={seedKey}` exactly.

### F9-4 — No file watcher; configs go stale silently
- **Category**: design / event-driven
- **Confidence**: high
- **Diff**: medium (~80 LOC across both layers)
- **Today**: If user edits `settings.json` from terminal (or our own discovery audit writes it), modal pane shows old text until reopened. The `notify` crate is already a backend dep (`observability/codex_rollout.rs:32`).
- **Proposed**:
  - Backend: new commands `watch_config_file(scope, working_dir, file_type) -> watch_id` + `stop_watching(watch_id)` using `notify::RecommendedWatcher`. Emit `config_file_changed` events.
  - Frontend: `useFileWatcher(path, onChange)` hook in `src/lib/`. The new `<TextFileEditor>` (F9-3) calls it. On change: if dirty, flash "File changed on disk" hint with Reload button (do NOT clobber buffer); if not dirty, reload silently.
- **Cross-platform**: `notify` works on Linux (inotify) and Windows (ReadDirectoryChangesW). No work for parity.

### F9-5 — Per-pane `setTimeout(() => onStatus(null), 2000)` repeats 20 times
- **Category**: simplify / leak
- **Confidence**: high
- **Diff**: small (-25/+20)
- **Today**: Every Save site re-implements "flash a status, then clear after 2s." Status banner already renders message in ConfigManager. Current code skips cancel-on-unmount cleanup, leaking timers.
- **Proposed**: `useFlashStatus` hook owning timer in one place, cancels on unmount. Replace all 20 sites.

### F9-6 — PromptsTab three independent feature groups should split
- **Category**: cohesion
- **Confidence**: low
- **Diff**: 0 net (~150 LOC moved)
- **Today**: 18 useState entries form three independent feature groups (My Prompts, Observed Prompts, Rules). PromptsTab is 742 lines but per-feature counts are roughly 200/200/200 + 50 wiring.
- **Proposed**: Pill-group + `<MyPromptsPane>`/`<ObservedPromptsPane>`/`<RulesPane>` switch (~60 lines). PromptsTab.tsx drops 742 → ~80; three new files ~200/220/250.

### F9-7 — Three localStorage `collapsed`-toggle copies
- **Category**: simplify / DRY
- **Confidence**: high
- **Diff**: small (-30/+10)
- **Location**: SettingsTab.tsx:43-53, EnvVarsTab.tsx:30-40, SettingsPane.tsx:131-142
- **Today**: Three copies of the same 12-line `useState` initialiser + toggle callback.
- **Proposed**: `useLocalStorageBoolean(key)` hook in `src/lib/`.

### F9-8 — Dead McpPane transport ternary
- **Category**: dead-code
- **Confidence**: high
- **Diff**: trivial (-10)
- **Location**: McpPane.tsx:441-451
- **Today**: Codex and Claude branches of transport dropdown produce identical option arrays. Ternary always returns the same three options.
- **Proposed**: Remove ternary.

## Cross-cutting concerns

- **F2 (settings store)**: F9-3's TextFileEditor extraction reduces bulk-destructure footprint.
- **R5 (proxy)**: F9-1 backend event-emit reuses existing patterns from `commands/process.rs:551`.
- **R2 (commands/config)**: F9-4's file watcher complements F9-1 by giving frontend deterministic config refresh.

## Out of scope (mentioned only because preamble suggests aggressive design)

- **"Reducer per pane"** — NOT recommended. State shapes are flat; transitions linear; useReducer adds boilerplate without correctness gain. The shared abstraction (F9-3's `<TextFileEditor>`) is what matters.
- **"Lifted form state"** — NOT recommended. McpPane's form belongs to McpPane; lifting forces ConfigManager to grow ~80 lines tracking 6 form scopes. Unsaved guard registry is the correct abstraction layer.
- **CLI duplication**: intentional per [DL-01]. Don't fight it.

## Risk and gating

- **TS/vitest gates**: F9-1/F9-2/F9-5/F9-7/F9-8 are pure-frontend, no schema changes.
- **Cargo gate**: F9-1 backend (event emit), F9-4 backend (notify watcher) — both reuse existing patterns. Risk: handle leaks if pane unmounts before `stop_watching`. Use same pattern as `codex_rollout::CodexRolloutHandle`.
- **NU-01 native-undo**: F9-3 must preserve `defaultValue + onInput + key={seedKey}` exactly. Vitest test asserting reseed semantics.

## Recommended sequencing

1. **F9-1** (poll → event) — single hard rule alignment, ~30 LOC. Highest priority.
2. **F9-2** (unsaved-guard coverage) — HooksPane data-loss is user-facing footgun.
3. **F9-5** (`useFlashStatus`) — small, deletes 20 sites, foundation for F9-3.
4. **F9-3** (`<TextFileEditor>`) — architecturally important, 100-200 LOC PR with NU-01 risk.
5. **F9-4** (file watcher) — complements F9-3 with event-driven refresh.
6. **F9-7** (`useLocalStorageBoolean`) — small DRY win.
7. **F9-8** (kill dead McpPane ternary) — hygiene.
8. **F9-6** (PromptsTab split) — lowest priority; mostly organizational.
