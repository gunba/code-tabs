# R1: discovery/claude.rs

## Files in scope
- src-tauri/src/discovery/claude.rs (2592 LOC; sync read + scan for Claude Code CLI binary, plus plugin/skill scanning, env var harvesting, settings schema mining)

## Findings

### F1: 5-step binary resolver lacks short-circuit on stale/duplicate hits and re-runs `read_if_exists` per branch
- **Category**: refactor
- **Confidence**: high
- **Diff size**: medium
- **Location**: src-tauri/src/discovery/claude.rs:25-175
- **Today**: Five `if let Some(content) = read_if_exists(...) { if is_claude_content { return Ok(...) } }` blocks repeat the same shape. Each candidate path is read fully (up to 500 MB) before content validation; the read happens ahead of any cheap early-out. There is no logging or visibility into which step actually fired. Step 4 invokes `npm root -g` synchronously on the UI thread.
- **Proposed**: Extract a typed enum `ClaudeSource { CmdShim, SymlinkTarget, DirectCli, SiblingNodeModules, LegacyVersions, NpmGlobal }` and a `Vec<Box<dyn Fn(...) -> Option<(PathBuf, &'static str)>>>` of candidate-path builders. A single loop calls `read_if_exists`/`is_claude_content` once per candidate and breaks on the first hit. Pair with F4 to drop the 500 MB cap.
- **Side effects**: source-string contract preserved.

### F3: `is_claude_content` is a brittle two-substring check that lets non-Claude bundles pass
- **Category**: correctness
- **Confidence**: medium
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:37-39
- **Today**: `content.contains(r#"name:""#) && content.contains(r#"",description:""#)`. Any minified JS with name/description object pair passes, including unrelated CLI bundles in the same node_modules tree.
- **Proposed**: Add a Claude-specific anchor: also require `"@anthropic-ai/claude-code"` or `"userInvocable:!0"` or `"getPromptForCommand"`. Two-line change.

### F4: 500 MB cap with full-buffer lossy-UTF8 decode for files known to be 5–60 MB
- **Category**: performance
- **Confidence**: high
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:27-34
- **Today**: 500 MB cap is meaningless for known-shape ~30MB binary. `String::from_utf8_lossy(&bytes).to_string()` reallocates and copies the entire buffer on any non-UTF8 byte.
- **Proposed**: Drop cap to 128 MB. Use `read_to_string` with `read` fallback wrapped via `Cow::into_owned`. When file is valid UTF-8 (it is — minified JS), no copy. Five-line change.

### F5: `.cmd` shim parser splits on `"` and accepts any segment ending `.js`
- **Category**: correctness
- **Confidence**: high
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:46-67
- **Today**: Crude split: any quoted token ending `.js` is accepted, even in comments or echo commands. Doesn't handle `.ps1` shims.
- **Proposed**: Regex `"([^"]+\.js)"` over whole shim contents. Five-line change.

### F6: regex compilation inside hot loops + duplicate `name_re.captures` on the same slice
- **Category**: performance
- **Confidence**: high
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:240-244, 336, 522, 680
- **Today**: Four `regex::Regex::new(...).unwrap()` calls inside `discover_builtin_commands_sync` per invocation; `strip_interpolations` recompiles per name; L246-247 reads cap[1] twice from regex state.
- **Proposed**: `once_cell::sync::Lazy<Regex>` for every regex defined inside discovery functions. Cross-cutting with codex.rs and discovery/mod.rs.

### F7: hardcoded namespace list at L705 is a scaling bomb
- **Category**: design
- **Confidence**: medium
- **Diff size**: medium
- **Location**: src-tauri/src/discovery/claude.rs:705-725
- **Today**: A `const NAMESPACES: &[&str]` of 19 hand-curated entries. Every new top-level settings namespace requires hand-editing.
- **Proposed**: Discover namespaces dynamically: `,([a-zA-Z][a-zA-Z0-9]{2,30}):<alias>\.object\(` shape, intersected with names whose body contains `.describe(`. Keep explicit denylist (already centralized in `is_settings_key_noise`).

### F8: `noise_keys` blocklist duplicated between L455-489 and L862-898
- **Category**: simplify (DRY)
- **Confidence**: high
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:451-491 vs 862-898
- **Today**: Two copies of the same `matches!` macro. Already factored as `is_settings_key_noise` — just unused at the primary site.
- **Proposed**: Replace L455-489 with `if is_settings_key_noise(&key) { continue; }`. Two-line change.

### F9: `process::Command::new("npm")` runs synchronously on the calling thread with no timeout
- **Category**: correctness / performance
- **Confidence**: high
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:148-172
- **Today**: `npm_cmd.output()` is blocking. On Windows, `npm.cmd` shells via `cmd.exe` (notoriously slow). On corporate machine with bad registry can hang tens of seconds. No timeout.
- **Proposed**: Skip npm fallback when paths 1-4 produced any readable file. Or add a hard timeout via tokio::time::timeout in caller.

### F10: silent `Err(_) => return Ok(Vec::new())` swallows actionable errors
- **Category**: correctness (observability)
- **Confidence**: high
- **Diff size**: small
- **Location**: src-tauri/src/discovery/claude.rs:229-232, 411-414, 1138 (similar)
- **Today**: When `read_claude_binary` fails, discovery returns empty `Vec` and discards reason. Audit/Settings UI can't distinguish "binary not found" from "found but rejected" from "found but empty after scan."
- **Proposed**: Propagate `Err(e) => Err(e)`. Two-line change per call site.

### F11: `walk_balanced` does not handle JS string literals — comment admits it
- **Category**: correctness
- **Confidence**: medium
- **Diff size**: medium
- **Location**: src-tauri/src/discovery/claude.rs:200-221
- **Today**: Comment at L196-199 says "stray `}` inside a quoted string can prematurely terminate the region." Used by three of four heaviest discovery paths.
- **Proposed**: String-aware lexer state machine: `enum LexState { Code, Single, Double, Tmpl, Regex, Comment }`. ~30-line replacement. Cross-cutting — belongs in `discovery/mod.rs`.

### F12: env var harvest passes 3-5 share regex shape but each constructs its own anchor list
- **Category**: simplify
- **Confidence**: medium
- **Diff size**: medium
- **Location**: src-tauri/src/discovery/claude.rs:1153-1258
- **Today**: Five sequential passes each iterate `content` independently. Pass 3, 4, 5 share the same `ENV_PREFIXES` allowlist and `near_env` proximity gate.
- **Proposed**: Single combined regex with alternation, single pass. Halves LOC and wall-clock.

### F13: `discover_plugin_commands_sync` walks dirs recursively without depth bound
