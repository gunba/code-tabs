# R3: commands/data.rs

## Files in scope
- `src-tauri/src/commands/data.rs` (1971 lines)

## Findings

### F1: Codec dispatch in `read_conversation_sync` is implicit type-tag matching, not a strategy pattern
- **Category**: design
- **Confidence**: high
- **Diff size**: medium
- **Location**: `data.rs:1829-1971` (`read_conversation_sync`); Codex/Claude branch arm at `1866-1887`
- **Today**: Single 142-line function does runtime branching on `parsed["type"]` to choose Codex vs Claude handling, falling through to Claude format if no Codex tag matches. Codex handlers (`codex_response_item_to_message`, `codex_compaction_marker`, `codex_text_from_content`) live as free functions; Claude inline-block parsing is hand-rolled inside the loop. No compile-time guarantee that all formats are handled, and a malformed Claude record is silently dropped.
- **Proposed**:
  ```rust
  trait ConversationCodec {
      fn classify(&self, line: &serde_json::Value) -> LineKind;
      fn parse_turn(&self, line: &serde_json::Value) -> Option<serde_json::Value>;
      fn extract_search_text(&self, line: &serde_json::Value) -> Option<(Role, String)>;
      fn enumerate_session_files(&self, root: &Path) -> Vec<SessionFile>;
      fn summarize_for_picker(&self, path: &Path) -> Option<RawEntry>;
  }
  struct ClaudeCodec; struct CodexCodec;
  ```
  Use the `enum SearchCli { Claude, Codex }` already at `data.rs:917` as the canonical discriminator.
- **Side effects**: `list_past_sessions_sync`, `search_session_content_sync`, `search_jsonl_files_sync`, `read_conversation_sync` all collapse onto codec methods.

### F2: `list_past_sessions_sync` is 255 lines mixing 4 separable concerns
- **Category**: refactor
- **Confidence**: high
- **Diff size**: medium
- **Location**: `data.rs:392-648`
- **Today**: One function does (a) directory walking, (b) head pass with UUID collection + first-message extract, (c) tail pass with model + last-message extract, (d) Codex branch, (e) chain detection. Two `std::fs::File::open(&fpath)` calls per Claude file (head L481 + tail L517) when one open + dual-direction reader would suffice.
- **Proposed**:
  ```rust
  fn scan_claude_session(path: &Path) -> Option<RawEntry>     // head + tail in one open
  fn scan_codex_rollout(path: &Path) -> Option<RawEntry>      // adapt summarize_codex_rollout
  fn resolve_chains(entries: &mut [RawEntry], uuid_map: ...)  // L631-642 lifted out
  ```
- **Side effects**: Local. Performance win on large `~/.claude/projects/` trees.

### F3: `search_session_content_sync` and `search_jsonl_files_sync` duplicate ~80% of their logic
- **Category**: refactor
- **Confidence**: high
- **Diff size**: medium
- **Location**: `data.rs:909-1076` and `1095-1252`
- **Today**: Two near-identical functions: walk projects dir, build file list with size/mtime gate, open each file, iter lines, run a regex/substring match, build a snippet. They differ in (a) input, (b) match semantics, (c) snippet construction. The Claude/Codex codec branch is forked between them.
- **Proposed**: Single `fn run_search(plan: SearchPlan, codecs) -> Vec<Hit>` that takes a `SearchPlan` describing source, matcher, aggregation. Removes ~167 duplicated lines.
- **Side effects**: Both Tauri commands become thin adapters.

### F4: Five overlapping text extractors
- **Category**: simplify
- **Confidence**: high
- **Diff size**: small
- **Location**: `data.rs:651-707, 761-803, 72-86`
- **Today**: Five free functions doing very similar work over `serde_json::Value` content arrays. They share the "pull text from content blocks" pattern but differ in filter/truncation policy.
- **Proposed**:
  ```rust
  fn collect_content_text(content: &Value, kinds: &[&str], joiner: &str) -> Option<String>
  fn extract_turn(line: &Value, codec: Cli) -> Option<(Role, String)>
  ```
- **Side effects**: Touches every search/picker path. Drops ~50 lines.

### F5: `truncate_preview` allocates `Vec<&str>` then full-size `String` before truncating
- **Category**: performance
- **Confidence**: high
- **Diff size**: small
- **Location**: `data.rs:63-70`
- **Today**:
  ```rust
  text.split_whitespace().collect::<Vec<_>>().join(" ").chars().take(limit).collect()
  ```
  For a 100KB rollout payload, the entire text is allocated even though only 150 chars are kept.
- **Proposed**: Single-pass writer that respects the char limit during accumulation.
- **Side effects**: Local. Reduces allocation pressure on resume picker.

### F6: Two passes over the tail in `list_past_sessions_sync`
- **Category**: performance
- **Confidence**: medium
- **Diff size**: small
- **Location**: `data.rs:526-554`
- **Today**: First forward pass collects UUIDs into the global map; second reverse pass extracts last user message + model. Both fully parse JSON.
- **Proposed**: One reverse pass that does both, early-exit when `last_msg + model` are populated.
- **Side effects**: Behavior change only if chain links resolve via tail UUIDs (audit needed).

### F7: `reveal_path` Linux branch ignores file-manager selection support
- **Category**: correctness
- **Confidence**: medium
- **Diff size**: small
- **Location**: `data.rs:191-219`
- **Today**: `[RC-22]` rule says "Linux uses `xdg-open <dir>`" but Linux file-managers all implement freedesktop `org.freedesktop.FileManager1.ShowItems`. Current code drops the user at the parent dir without highlighting the target file.
- **Proposed**: Try `dbus-send --session ... ShowItems` first, fall through to `open_path(parent)`.
- **Side effects**: Linux only. Per CLAUDE.md rule 3, flag for user approval before deletion of the bare-fallback path.

### F8: `codex_id_from_rollout_filename` validates UUID by counting hyphens — fragile
- **Category**: correctness
- **Confidence**: medium
- **Diff size**: small
- **Location**: `data.rs:42-51`
- **Today**: Counts dashes but doesn't validate hyphen positions or hex characters.
- **Proposed**: Precise regex with `OnceLock`:
  ```rust
  static UUID_RE: OnceLock<Regex> = OnceLock::new();
  let re = UUID_RE.get_or_init(|| Regex::new(
      r"([0-9a-fA-F]{8}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{12})$"
  ).unwrap());
  ```
- **Side effects**: Local.

### F9: Codex content joiner uses `"\n"`; Claude joiner uses `" "` — silent format drift
- **Category**: correctness
- **Confidence**: high
- **Diff size**: small
- **Location**: `data.rs:781-786` (Codex) vs `data.rs:702-704` (Claude)
- **Today**: Same conceptual operation, different separator. Search snippets across CLIs become inconsistent.
- **Proposed**: Pick one (`"\n"` is more faithful) and document. Centralize in F4's `collect_content_text`.
- **Side effects**: Search match counts may shift slightly. Update tests.

### F10: `MAX_CONVERSATION_MESSAGES = 2000` and `MAX_CONTENT_SEARCH_FILE_SIZE = 20MB` truncate silently
- **Category**: tech-debt
- **Confidence**: high
- **Diff size**: small
- **Location**: L906, 1675-1677
- **Today**: Files >20MB silently dropped; conversations >2000 messages silently truncated. User has no signal.
- **Proposed**: Emit a structured event via `observability` module, or push a sentinel:
  ```rust
  if messages.len() >= MAX_CONVERSATION_MESSAGES {
      messages.push(serde_json::json!({
          "role": "system",
          "content": [{ "type": "truncation_notice", "limit": MAX_CONVERSATION_MESSAGES }],
      }));
      break;
  }
  ```
- **Side effects**: Frontend touches — F8/F10 reviewers need to render the sentinel.

### F12: `cleanup_session_data` swallows manifest parse errors
- **Category**: simplify
- **Confidence**: high
- **Diff size**: small
- **Location**: `data.rs:281-310`
- **Today**: Five-level nested `if let / else` ladder. If anything fails, the dir is *not* removed — losing the documented "fall back to dir mtime" behavior.
- **Proposed**:
  ```rust
  fn dir_started_at(dir: &Path) -> Option<SystemTime> {
      let manifest = std::fs::read_to_string(dir.join("manifest.json")).ok()?;
      let val: serde_json::Value = serde_json::from_str(&manifest).ok()?;
      let started = val["startedAt"].as_str()?;
      chrono::DateTime::parse_from_rfc3339(started).ok().map(SystemTime::from)
  }
  ```
- **Side effects**: Local. Behavioral fix.

## Cross-cutting concerns

- **Codec trait propagates to F2/F3/F4/F9** — adopting strategy pattern is the single biggest leverage point.
- **`MAX_*` constants belong on the codec impls** — Claude's 20MB cap is unrelated to Codex's.
- **`spawn_blocking` boilerplate** wraps five Tauri commands identically.
- **No telemetry** for silent-skip cases — user CLAUDE.md says "implement telemetry first".
- **Path normalization is ad-hoc** at L436, L595, L1177. Centralize in `path_utils`.
- **Cross-unit overlap**: F8 (ResumePicker) consumes `list_past_sessions`, F10 (small components) consumes `read_conversation`.
