## Findings

### F1: Inconsistent atomicity — three writers skip the temp+rename pattern
- **Category**: correctness
- **Confidence**: high
- **Diff size**: small (<50)
- **Location**: src-tauri/src/commands/config.rs:151, :831, :1188 (and the create_dir_all+write at :140/:1847 wrappers)
- **Today**: `save_hooks` (`std::fs::write(&settings_path, output)`), `write_config_file` (`std::fs::write(&path, &content)`), and `save_event_kinds` (`std::fs::write(&path, json + "\n")`) all do non-atomic writes on the exact same files (`~/.claude/settings.json`, `~/.codex/config.toml`, `~/.claude/CLAUDE.md`) that `write_codex_config_value` and `write_mcp_servers` already protect with `atomic_write`. The comment at L1884 explicitly justifies atomicity for `~/.claude.json` because Claude Code CLI writes it concurrently — settings.json has the same contention surface.
- **Proposed**: route every persistence path through `atomic_write` (L1889, Windows/Linux safe via same-directory temp + `std::fs::rename`). Replace the three `fs::write` call sites with `atomic_write(...).map_err(...)`. Centralize `create_dir_all(parent)` into the helper as a single primitive.
- **Side effects**: Pure win on partial-write resilience. No signature changes, no test breakage.

### F2: `resolve_api_host` (L1920) is cosmetic — kill it, don't optimize it
- **Category**: dead-code (effectively)
- **Confidence**: high
- **Diff size**: medium (50-300, including frontend)
- **Location**: src-tauri/src/commands/config.rs:1916-1946 (~31 lines), src/hooks/useTapEventProcessor.ts:877-883, src/components/StatusBar/StatusBar.tsx:115/132-134, src/store/settings.ts:241/289/347/830
- **Today**: 27-line async Tauri command does a 5-second `tokio::time::timeout` around `spawn_blocking(ToSocketAddrs)`, fires once on the first `ApiFetch` event per session, and the only consumer is StatusBar.tsx which renders it as a tooltip string `IP: 1.2.3.4`. Result is purely informational — nothing branches on it. The `apiIpResolvedRef.current = true` gate is itself a hand-rolled cache (the user rule forbids caching).
- **Proposed**: delete `resolve_api_host`, `apiIp`/`setApiIp` from settings store, the resolve-on-ApiFetch block, and the StatusBar tooltip. If the IP is genuinely valuable, surface it from `src-tauri/src/proxy/` where the socket peer is already known. The right perf optimisation is to not do the work at all.
- **Side effects**: Cross-unit flag F2 (settings store), F7 (StatusBar), F5 (tap pipeline). Removes one entry from R10's lib.rs handler list.

### F3: `resolve_config_path` (L672-758) is a 90-line catch-all — split into a strategy enum
- **Category**: design
- **Confidence**: high
- **Diff size**: medium (50-300)
- **Location**: src-tauri/src/commands/config.rs:672-758
- **Today**: One function with eight match arms, four using `_ if file_type.starts_with(...)` guards (`agent:`, `agent-delete:`, `skill:`, `skill-delete:`, `codex-skill:`, `codex-skill-delete:`). Each prefix variant repeats the same `split_once(':')`, `validate_md_file_name`, scope-match-to-`(home, project)` skeleton. Same scope→base-dir mapping is re-derived in `cli_skill_source_roots` (L906), `cli_skill_dest_root` (L934), `list_agents` (L1232), `list_skills` (L1303), `list_codex_skill_files` (L1351), and `resolve_activity_context_files` (L1646).
- **Proposed**: extract a `ConfigKind` enum with one match producing a `ResolvedPath`. Each variant owns its resolver:
  ```rust
  enum ConfigKind { Settings, ClaudeMd(Scope), CodexConfig, AgentsMd(Scope),
                    Agent { name: String }, Skill { kind: SkillKind, name: String }, ... }
  fn config_root(scope: Scope, working_dir: &Path, cli: Cli) -> PathBuf
  ```
  `resolve_config_path` becomes parsing-only (`parse_file_type` returns `ConfigKind`); downstream callers consume `config_root` directly. Validation moves into the ConfigKind constructor.
- **Side effects**: Internal-only — keeps the file_type wire format identical for F9 ConfigManager panes.

### F5: Hooks/MCP/Skills/Plugins discovery duplicate scan-then-aggregate scaffolding
- **Category**: refactor
- **Confidence**: medium
- **Diff size**: medium (50-300)
- **Location**: discover_hooks (L60-113), discover_codex_hooks (L304-352), list_codex_skill_files (L1346-1382), list_agents (L1227-1254), list_skills (L1298-1340), read_codex_plugins (L1090-1097)
- **Today**: Each command does (1) resolve scope-keyed directories, (2) `read_dir + filter`, (3) build a `serde_json::Map<String, Value>` keyed by scope-prefix, (4) emit a `log_discovery(...)` event with the same shape (workingDirs, scannedFiles, scopeCount, scopes/names/count). Six near-identical `json!({...})` blocks.
- **Proposed**: introduce a `DiscoveryReport<T>` aggregator:
  ```rust
  struct DiscoveryReport<T> {
      scanned_files: Vec<PathBuf>,
      scopes: BTreeMap<String, T>,
  }
  impl<T: Serialize> DiscoveryReport<T> {
      fn log(&self, app: &AppHandle, event: &str, message: &str)
      fn into_value(self) -> serde_json::Value
  }
  ```
  Each command becomes 5-10 lines.
- **Side effects**: Internal-only — wire format unchanged.

### F6: `write_config_file` validation conflates "invalid input" with "I/O failure" — typed error
- **Category**: design
- **Confidence**: medium
- **Diff size**: small (<50)
- **Location**: src-tauri/src/commands/config.rs:815-832
- **Today**: Validation errors and I/O errors both come back as `String`. Frontend can't distinguish "your TOML is malformed" from "disk full / permission denied".
- **Proposed**:
  ```rust
  #[derive(serde::Serialize)] #[serde(tag = "kind")]
  pub enum ConfigError {
      InvalidInput { message: String },
      InvalidScope { scope: String },
      MissingWorkingDir { scope: String },
      Io { path: String, message: String },
      ParseExisting { path: String, message: String },
  }
  ```
  Tauri serializes via Serialize directly — frontend `invoke` rejects with a typed object.
- **Side effects**: Cross-unit flag F9. Migration is incremental.

### F7: `merge_hook_values` is O(n*m) AND `save_codex_hooks` rewrites hooks.json to `{}` to compensate
- **Category**: design / correctness
- **Confidence**: high
- **Diff size**: small (<50)
- **Location**: src-tauri/src/commands/config.rs:254-276, :390-394
- **Today**: `merge_hook_values` (L254) appends each hook from `incoming` only if `!existing.iter().any(|e| e == hook)` — quadratic JSON-value-equality scan. `save_codex_hooks` (L390) then rewrites `hooks.json` to `{}` after each save because otherwise save→reload doubles entries. The known bug is shoved into the saver instead of fixed at merge time.
- **Proposed**: hash each hook entry with a stable canonical-form serialization into a `HashSet<String>` once, push if `seen.insert(canonical)`. Linear instead of quadratic. Lets `discover_codex_hooks` reconcile sources without `save_codex_hooks` having to nuke `hooks.json`.
- **Side effects**: Behavioral change — preserves user-authored `hooks.json` content across UI saves. Worth verifying with the user that hooks.json wipe is intentional.

## Cross-cutting concerns

1. **Atomicity is not consistent across writers.** F1 fixes it; broader principle is to codify "every persistent file write goes through `atomic_write`" via a `WriteTarget` newtype.
2. **Scope handling is duplicated everywhere.** 13 occurrences. F4+F8 fix locally; deeper fix is a `Scope` enum.
3. **Error type discipline.** Every command returns `Result<_, String>`. F6 proposes a typed error.
4. **Discovery commands all log the same shape.** F5+F10 unify them.
5. **Read-merge-write closure helper is half-applied.** F11 generalizes.

## Out of scope

- `atomic_write` panic safety: temp file leaks if killed between write and rename.
- `codex_spawn_env_file` `project_hash` collision risk: 64-bit truncated SHA256.
- `resolve_skill_file_inner` recursive search depth (8) and visited-cap (4000): bounded and well-commented.
- `expand_rule_reference` `~/`-prefix handling: POSIX convention.
- `copy_dir_recursive`: bounded by directory size, no symlink-loop guard.
