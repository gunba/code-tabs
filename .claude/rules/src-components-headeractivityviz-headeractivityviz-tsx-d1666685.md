## Header Activity Visualizer
Ambient soundwave-and-mascots scene that fills the gap between the tab strip and the right-side action buttons

- [HA-01 L12] HeaderActivityViz paints the slack between .tab-bar-scroll and the right-hand action buttons (.tab-resume / .tab-config / .tab-add) into a single <canvas> as a pixel-art / 'voxel' coastal scene — sky, sun/moon, weather, layered ocean with foam, textured beach with decorations, and animated agent mascots. Backing store: dpr*cssSize with ctx.scale(dpr,dpr) and imageSmoothingEnabled=false for HiDPI crispness; ResizeObserver on the wrapper triggers needsResize and MutationObserver on document.documentElement style-attribute bumps themeBumpRef so theme/css-var changes invalidate cached tiles. All paint runs in one rAF tick — pre-rendered atlases (mascot sprites, subagent icons, sand tile, decoration tile) keep per-frame cost cheap. Layout via computeLayout(w,h): beachW = clamp(round(w*0.13), 96, 180); seaMeanY = round(h*0.6); beachShoreSlope = max(9, round(h*0.18)); waveAmpMaxPx = max(5, round(h*0.13)); beachTopYAtZero = seaMeanY - beachShoreSlope; skyHorizonY = round(h*0.45). Beach surface via shoreYAt(layout, tt) with tt in 0..1 — quick rise from elev=0.65 inland edge to elev=0.95 plateau (tt=0.15-0.45), then ONE continuous smoothstep descent from elev=0.95 to elev=-2.0 across tt=0.45..1.0; the surface crosses sea level (elev=0) around tt=0.648 and dips below the canvas bottom around tt=0.91 for thin canvases (h<44), so the bank disappears smoothly under the water rather than terminating in a vertical cliff. surfaceY = seaMeanY - elev*beachShoreSlope. Wave crests via computeWaveCrests on a shared Float32Array — two sin waves (period 110/47, speed 36/22), amplitude scaled by intensity (0.45..1) and ramped from 0 at x=beachW over rampDist = max(20, round(waveAmpMaxPx*3)) so sea joins beach without a step. Render pipeline: (1) drawSky — base gradient by scene + horizon glow + day/night tint via celestialPhase(sunriseHour, sunsetHour) + uniform vertical backlight gradient over the upper sky (cool 180,200,235@0.18 alpha at night when moon is up; warm 255,240,200@0.10 on clear/cloud/snow daytime with phase.light>0.55; suppressed in fog/storm) so the moon/sun reads as the actual light source rather than a sticker on a flat sky; (2) drawStars at night when not fog/storm and nightAlpha > 0.15; (3) celestial body — phase.isNight ? drawMoon(cx=w-36, cy=22, r=11) : drawSun(cx=w-32, cy=18, r=8) drawn unconditionally so the sky is never empty; (4) drawClouds(ctx, cloudsRef, w, dtSec, scene, t) when scene is clouds/rain/storm — paints a continuous drawOvercast ceiling first for rain/storm (gradient band along canvas top with bottom edge wobbled by three layered sines), then 4/5/6 discrete Simpsons-style cumulus puffs (count by scene; widths 48/74/100 px cycling by i%3) drawn as a UNION OF OVERLAPPING FULL-CIRCLE SUBPATHS (moveTo + arc(0,2π) per bulge, single fill with nonzero winding so alpha doesn't double up at overlaps) — three layout variants of 4-6 bulges with radii 5-12 px and outer tRel inset (0.12..0.88) so cx±r stays within [x, x+w] for the narrowest cloud; (5) drawFog when scene==='fog'; (6) drawSeagulls on the daylight side, skipped in storm/snow/fog; (7) drawSea — gradient sea polygon starts inland at round(beachW*0.55) so it covers the underwater bank zone where shoreYAt now traces below seaMeanY (without this the gap between disappearing sand and beachW would show theme.bgSurface) + bright crest band + sparkles (clear/clouds) + foam line along the wave crest from x=beachW to w; (8) drawMoonReflection + drawNightSeaSparkles when night and scene is clear/clouds/snow; (9) drawBeach — clipped tiled sand + decoration tile painted along the curved dune top; the main polygon's right edge dips below sea level via the new shoreYAt and tapers off-canvas, so no separate underwater-bank overlay is drawn; then drawBeachLandmarks (drawUmbrella at tt=0.30 on the plateau, drawFoldoutChair at tt=0.55 on the upper seaward descent — back leg at tt-0.04, front leg at tt+0.04 anchored to shoreYAt so the seat tilts with the local slope); (10) drawShoreChop — foam band gated to columns where bankY is within ±wlBand=max(2, round(waveAmpMaxPx*0.5)) of seaMeanY (so foam only paints where the sand surface actually crosses the waterline, never tracing diagonally up the dry slope); loop iterates from round(beachW*0.5) and breaks once bankY>seaMeanY+wlBand+2 (relies on shoreYAt being monotonically non-decreasing past the plateau); bursting whitecaps anchored at xPlume = beachW*0.55 + ((seed>>>8) % round(beachW*0.15)) — tt range [0.55, 0.70] aligned with the new waterline; (11) drawSlots — agent mascots tracked in slotsMapRef Map<id, Slot> with xPx, dir, homeXPx/homeRow01, speedPxPerS, jitterSeed, diveT, bubbleAccumMs; sprites are claude-mascot.png / codex-mascot.png pre-rasterized at MASCOT_PX=22 (subagents SUBAGENT_PX=14) plus AgentTypeIcon SVG rasterized via svgToImage; active agents bounce between layout.beachW+4 and w-MASCOT_PX, idle agents rest on the beach at hash-determined homeXPx/homeRow01 (BEACH_PAD_PX=4), errored agents dive (diveT 0..1) and emit bubbles; (12) bubble trails — BUBBLE_INTERVAL_MS=320ms cadence, BUBBLE_LIFE_MS=1200ms lifetime, drifting up at 22 px/s; (13) foreground rain/snow particles via dropsRef/flakesRef; (14) storm bolt + flash every STORM_PERIOD_S=7s (STORM_FLASH_S=0.12s, STORM_BOLT_S=0.18s). makeBeachDecoTile scatters pebbles/shells/conches/starfish/driftwood and a 5-step footprint trail clamped to dryW = max(8, round(beachW*0.6)) — sea level lands around tt=0.65 and the 0.6 cutoff leaves a ~5% dry-sand margin so decorations never end up underwater. cloudsRef/flakesRef/dropsRef/seagullsRef are all cleared in a single useEffect on weatherCode change so a scene transition doesn't carry over stale particle counts. Activity intensity samples session.metadata.toolCount deltas every 500 ms via useSessionStore.getState() (no subscription, no per-frame re-render); burst = 1-exp(-delta/3); EMA intensityRef.current * INTENSITY_DECAY_RETAIN_PER_500MS=0.7 + (BASE_INTENSITY=0.18 + 0.85*burst)*0.3 drives wave amplitude scaling; lastCounts seeds new sessions so historical activity isn't replayed as a burst. Component is aria-hidden, pointer-events:none — pure decoration. CSS shrinks to ~20 lines: .header-activity-viz (flex:1 1 0; min-width:0; overflow:hidden; pointer-events:none; border-left/right:1px solid var(--border-subtle)) wrapping a .header-activity-viz-canvas (display:block; width:100%; height:100%). Pure exported helpers (hash32, makeSlotInit, withAlpha, clampPx, shoreYAt) plus celestialPhase are unit-tested via __tests__/HeaderActivityViz.test.ts (15 cases — hash32/makeSlotInit/withAlpha/clampPx plus three shoreYAt cases for waterline crossing, exact seaward-end value seaMeanY+2*beachShoreSlope, and monotonic descent on tt∈[0.45,1.0]).
