Tauri v2 desktop app managing multiple Claude Code and OpenAI Codex CLI sessions in tabs. Rust backend + React/TypeScript frontend. No API key — uses installed CLIs directly.

# Workflow

The user launches with `claude -w` for isolated worktrees. Shared directories such as `.claude/rules/` are symlinked into those worktrees.

The user will run `/r`, `/j`, `/b` when ready. Do not run these automatically.

# Documentation

Do not hand-edit `.claude/rules/*.md`; they are generated by an external system and refreshed during release or finalization.

# Discovery Audit

Claude settings, slash commands, and env vars in the app are extracted from the Claude Code binary by regex scans in `src-tauri/src/discovery/`. Codex models/options/commands are discovered separately from the installed Codex CLI and binary where available. These scans are fragile, so binary or CLI output changes can silently drop items. When anything feels missing from the Settings modal or Command palette, run the relevant audit or discovery dump.

The audit runs discovery once and diffs the output against cached copies of `code.claude.com/docs/en/{settings,commands,env-vars}` (checked into `src-tauri/tests/fixtures/`). It reports items docs list but discovery did not find.

```bash
# Dump what discovery currently finds (no network, no diff).
npm run discover:dump -- --what commands --format pretty
npm run discover:dump -- --what settings --format pretty
npm run discover:dump -- --what env-vars --format pretty

# Refresh docs fixtures before an audit. Only run when the docs site has changed
# or when the current fixtures are stale — not on every audit.
npm run discover:fetch-docs

# Full audit (exits non-zero on missing items).
npm run discover:audit
npm run discover:audit -- --format pretty  # human-readable summary
```

Output semantics:
- `missing` = documented but not discovered. Likely a real gap in our regex.
- `extra` = discovered but not documented. Usually OK — the user may have a newer binary than the cached docs, or the docs omit internal keys.

When the audit reports new `missing` items, the fix goes in `src-tauri/src/discovery/mod.rs` (not in the static fallback registry — tighten the scanner so the next version works too). Tests alongside each fix live in the same file.

CI runs the audit automatically on PRs that touch `src-tauri/src/discovery/**`, `src-tauri/src/commands/cli.rs`, or the docs fixtures, and weekly on schedule. It is advisory only — drift is surfaced, not blocking.

Do not add docs-download logic to the runtime app. Fetching lives only in the `discover_audit fetch-docs` subcommand and is invoked by the developer or CI.
