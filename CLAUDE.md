- For anything else, ask the user to log some test data. Logs are at [AppData\Local\claude-tabs\sessions], depending on what logging the user has enabled. You can review these using the debug skill, ideally with a subagent.

Tauri v2 desktop app managing multiple Claude Code CLI sessions in tabs. Rust backend + React/TypeScript frontend. No API key — uses the Claude Code CLI directly.

# Workflow

The user launches with `claude -w` for isolated worktrees. Shared directories such as `.claude/rules/` are symlinked into those worktrees.

The user will run `/r`, `/j`, `/b` when ready. Do not run these automatically.

# Documentation

Do not hand-edit `.claude/rules/*.md`; they are generated by an external system and refreshed during release or finalization.
